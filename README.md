# 发车日记 (train-departure-diary)

一款面向家禽交易行业的移动端管理应用，用于管理鸡场交易、发车记录、员工结算等业务流程。

## 技术栈

| 类别     | 技术              |
| -------- | ----------------- |
| 框架     | UniApp (Vue 3)    |
| 状态管理 | Pinia             |
| 数据库   | 自建阿里云 MySQL  |
| 后端     | Express (Node.js) |
| API      | HTTP REST API     |
| 图表     | uCharts           |
| UI 组件  | uni-ui            |

## 功能模块

### 核心功能

- **首页** - 报价日历、图表展示、今日发车记录
- **发车管理** - 发车记录的新增、编辑、删除
- **鸡场管理** - 鸡场信息的 CRUD 操作
- **员工管理** - 员工信息的 CRUD 管理
- **交易记录** - 交易流水记录
- **结账统计** - 员工/鸡场结账统计
- **盈利统计** - 利润分析
- **个人中心** - 用户设置、邀请码管理

### 角色体系

| 角色   | 说明                          |
| ------ | ----------------------------- |
| 管理员 | 可见全部数据，按中间商维度展示 |
| 中间商 | 可见自己+下属数据            |
| 装发车 | 可见自己的发车记录          |
| 鸡场   | 无发车记录权限              |

## 项目结构

```
├── pages/               # 页面组件
│   ├── login/           # 登录页
│   ├── home/            # 首页
│   ├── departure/       # 发车管理
│   ├── merchant/        # 鸡场管理
│   ├── worker/          # 员工管理
│   ├── transaction/    # 交易记录
│   ├── statistics/      # 结账统计
│   ├── user/            # 盈利统计
│   ├── profile/         # 个人中心
│   └── admin/           # 超管页面
├── store/               # Pinia 状态管理
├── utils/               # 工具函数
├── components/          # 公共组件
├── static/              # 静态资源
├── sql/                 # SQL 脚本
└── uni_modules/         # uni-ui 插件
```

## 开发指南

### 环境要求

- Node.js v20+
- HBuilderX (推荐) 或 VS Code
- JDK 17 (用于 Android 构建)

### 安装依赖

```bash
npm install
```

### 运行项目

使用 HBuilderX 打开项目，选择运行到 Android 或 H5。

### 后端服务

后端已独立部署，API 请求发送至: `http://47.96.90.103:3000/api`

本地后端启动:
```bash
cd ~/Desktop/train-departure-diary/train-departure-diary-server && node index.js
```
